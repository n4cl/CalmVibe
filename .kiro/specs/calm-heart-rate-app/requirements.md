# Requirements Document

## Introduction
急激な心身の高ぶりを鎮めるスマホアプリMVPとして、呼吸ガイドとリズム振動による鎮静体験を提供し、心拍は手入力でセッション前後を記録・振り返りできる（心拍データの自動計測・同期は行わない）。

## Requirements

### Requirement 1: 個人設定
**Objective:** ユーザーとして、事前に自分に合うガイド感覚を設定しておきたい。なぜなら、発作時に迷わず使いたいから。

#### Acceptance Criteria
1. 設定画面で、呼吸ガイドのテンポ（例: 4-6-4秒、5-5-5秒など複数プリセット）を選択・保存できること。
2. 設定画面で、バイブレーションの強さとリズムパターンを選択・保存できること。
3. 設定変更後、プレビュー（短いデモの音／振動）を再生し体感を確認できること。
4. 保存した設定は次回以降のセッション開始時に自動で反映されること。

### Requirement 2: リズム振動と呼吸ガイド提示
**Objective:** ユーザーとして、最小操作で呼吸/リズムガイドを受けたい。なぜなら、パニック時に落ち着くまでの手順をシンプルにしたいから。

#### Acceptance Criteria
1. ユーザーが「鎮静開始」をタップしたとき、ガイドモード（呼吸ガイド／リズム振動のみ）を選択させること。
2. セッション開始時に、呼吸ペース（例: 4-6-4秒）を視覚と音または振動で提示すること。
3. セッション進行中は、デバイス振動をガイドリズムに同期させ、無音モードでも振動のみでガイドを継続すること。
4. ユーザーが「静音モード」を選択した場合、音声を停止し振動のみでガイドすること。
5. セッション時間が終了したとき、ガイドを自動停止し終了メッセージを表示すること。

### Requirement 3: セッション記録と振り返り
**Objective:** ユーザーとして、セッション前後の状態や体感を記録して後から振り返りたい。なぜなら、効果を把握し次回に活かしたいから。

#### Acceptance Criteria
1. セッション開始前に、現在の心拍数を手入力で記録できること（未入力の場合は記録不要として進行できること）。
2. セッション終了直後に、終了時の心拍数を手入力で記録できること。
3. セッション終了時に、使用したガイド種別（リズム振動／呼吸ガイド）を選択して保存できること。
4. セッション終了時に、体感（落ち着き度合い、ガイドの心地よさ、振動の強さの感じ方など）を3〜5段階で入力して保存できること。
5. セッション終了時に、改善度合いを5段階で評価して保存できること。
6. 記録したセッション履歴を一覧で確認でき、各セッションの開始前後心拍・ガイド種別・体感入力・改善度合いが閲覧できること。
